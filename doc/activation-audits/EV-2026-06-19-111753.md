## Why ship-now (not park)

The prior audit `EV-2026-06-11-200211` parked the repo because no shipped-test
evidence was *documented* — not because the project was inert. Inspection
confirms a coherent, installable Python package (`hierarchia/`): Pydantic models
→ loader → registry → executor → validator, exercised by a pytest smoke suite
(`tests/test_smoke.py`). Its only runtime dependency is `pydantic>=2`. The
blocker was activation infrastructure, not code. This audit ships that
infrastructure so the execution path is durable and reviewer-verifiable.

## Reproduce from a clean checkout

```bash
git clone https://github.com/organvm-i-theoria/hierarchia-mundi
cd hierarchia-mundi
python -m venv .venv && . .venv/bin/activate
pip install -e ".[dev]"
ruff check .
pytest -q
```

Programmatic smoke check (load all strata, search, filter, validate):

```python
from hierarchia.loader import load_hierarchia
from hierarchia.registry import get_registry
from hierarchia import ModuleType, Scale
from hierarchia.validator import validate_hierarchy

h = load_hierarchia(".")
print(h.stratum_count, "strata,", h.module_count, "modules")
print(len(h.search_modules("entropy")), "entropy matches")
reg = get_registry(".")
print(len(reg.by_type(ModuleType.LAW)), "laws;",
      len(reg.by_scale(Scale.COSMIC)), "cosmic modules")
report = validate_hierarchy(".")
print(report.files_parsed, "files parsed;",
      len(report.errors), "errors;", len(report.warnings), "warnings")
```
